# Security Policy

The Encatch team takes the security of the Encatch Flutter SDK seriously.

## Supported Versions

Security updates are provided for the latest published version of the SDK. We recommend keeping your application updated to the newest available release.

| Version | Supported |
| ------- | --------- |
| Latest published version | Yes |
| Older versions | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

To report a vulnerability, contact the Encatch team privately at:

security@encatch.com

Include as much detail as possible:

- A description of the vulnerability
- Steps to reproduce or proof-of-concept details
- Affected SDK version or commit
- Affected platform, such as Android, iOS, macOS, or Windows
- Any known impact or mitigation

Do not include customer data, production API keys, or unrelated personal information.

## Response Process

We will review security reports and respond as soon as possible. If the issue is confirmed, we will work on a fix, publish an updated SDK version, and document relevant remediation guidance.

## Disclosure

Please give us a reasonable opportunity to investigate and resolve confirmed vulnerabilities before public disclosure.
